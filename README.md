# PokeMongoose 
Welcome Trainers!

Here is a bunch of starter routes and Models to interact with the Pokemon Database that I listed on Slack.
## Pokemon
There are some very basic GET and POST routes, and some entirely unfinished DELETE and PATCH routes.

Listed in the comments are some additional functionality to add! 

## Trainers
The model is missing a virtual property for the Average Level of the Pokemon for that trainer.

Additionally, there are the same basic GET and POST routes, and unfinished DELETE and PATCH routes.

Listed in the Comments are some additional functionality to add!

## Types
The model for Types is complete, and the Database has all the information that you need to interact.

You can add some additional routes here to provide useful information. Maybe a route with 2 route parameters, that tells you what types of attacks would work best on that type of pokemon!

# What to do with this Repo
1. Make a fork of this repo to your GitHub account
2. Clone that repo to your computer using `git clone link_to_your_repo` in the terminal!
3. create a .env file matching the template of the .env.example, but fill in the Database link from slack. Don't forget to add pokemon between the / and the ? or else it will not connect!
4. As you make changes and commits, push them to your repo! Send me a link on Slack if you gave it a shot!
